### Detailed recommendations

#### [Recommendation #1]: Auto-generate directory tree in README

**What**: Replace the hand-written ASCII tree in README.md with an
auto-generated one.

**How**:
1. Add a `<!-- tree:start -->` / `<!-- tree:end -->` marker pair in README.md.
2. Create a script `scripts/update-readme-tree.sh`:
   ```bash
   tree -L 2 -I 'node_modules|.git|dist|build' --charset=ascii > /tmp/tree.txt
   sed -i '/<!-- tree:start -->/,/<!-- tree:end -->/{ /<!-- tree/!d; /<!-- tree:start/r /tmp/tree.txt' }' README.md
   ```
3. Add to CI: run the script and `git diff --exit-code README.md` to fail
   the build if the tree is stale.

**What it replaces**: The current hand-maintained tree block (lines 42-67).

**Drift risk if not automated**: The tree diverges every time a directory is
added, renamed, or removed. Given the repo's refactor frequency (~2/month),
expect drift within 2-4 weeks.

## Notes on adapting the template

- If the user has formatting preferences (e.g., frontmatter disclaimer),
  incorporate them into the header.
- If the repo is small (<10 doc files), collapse the inventory and sync
  tables into a single section.
- If the repo is a monorepo, consider producing per-package sections or
  a separate report per workspace.
- Always preserve the methodology section — it is epistemic hygiene.
- The quarantine proposals table must include the TL;DR column — this is
  the whole point of not just deleting things silently.
